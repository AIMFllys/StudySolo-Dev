# Workflow Canvas 架构指南

本文档记录了工作流画布（Canvas）内部的核心操作流水线设计，便于后续持续迭代（特别是即将加入的节点间连线等功能）。

## 一、操作流水线 (History Pipeline)

为了提供符合行业标准的体验，我们实现了一套基于 Zustand 内存状态的历史操作栈，用于支持**撤销 (Undo)** 和**重做 (Redo)**。

### 1. Snapshot (快照) 机制
历史记录的粒度为整个画布的状态（`nodes` 和 `edges` 数组），每次触发时会完整保留一份拷贝：
- **容量上限**：最多保留 `50` 步历史（行业推荐标准），以防止内存溢出。
- **记录时机**：并不是每次 `x/y` 只要变动一点就记录，而是：
  - **节点拖拽结束**：在 `<ReactFlow>` 绑定的 `onNodeDragStart` 事件中保存移动前的状态。由于位置移动在内部是由持续的变更派发的，这就确保了一次移动（不论多久距离）合并为一次操作。
  - **节点/边删除**：我们通过拦截 `onNodesChange` 和 `onEdgesChange` 里的 `remove` 事件，将变动前状态推入 `past`。
  - **节点/边连接**：我们在 `onConnect` 中记录添加连线前的一刻。
  - **节点粘贴**：在剪贴板粘贴（添加一组节点）发生时主动调用 `takeSnapshot()` 保护此前状态。

### 2. 同步与隔离
- **物理按键**：原生地绑定了 `Ctrl+Z` / `Cmd+Z` 以及 `Ctrl+Y` / `Cmd+Shift+Z`。
- **与云端解耦**：当触发 `undo()` 改变界面时，内部由于调用了统一的 `isDirty=true`，会自动复用现有的防抖机制（`use-workflow-sync`），将恢复后的状态平滑同步回 IndexedDB 和 Supabase，无需额外的 API 代码封装。

## 二、剪贴板引擎 (Clipboard Strategy)

我们摒弃了纯内存（In-Memory）的节点复制机制，采用了更现代沉浸的系统剪贴板（`navigator.clipboard`）：

### 机制与设计取舍
1. **序列化**：当用户选中多个节点按下 `Ctrl+C` 时，应用会将所有选中节点转化为 JSON 字符串（带有 `source: 'studysolo-canvas'` 指纹），写入用户的物理剪贴板。
2. **跨页面/跨域优势**：借由系统剪贴板支持跨标签页共享节点的代码片段分享，用户甚至可以将 JSON 粘贴在文本编辑器中存档。
3. **坐标投影计算**：
   - 粘贴的目标区域必须**完全跟随用户的鼠标位置**（而不是偏移固定 50px）。
   - 实现方案为：我们在画布最外层侦听 `mousemove` 保持实时追踪真实的 `clientX/clientY`；
   - 粘贴触发时利用内部提供的 `reactFlowInstance.screenToFlowPosition`，反算并归一化偏移量，将粘贴节点的中心放置于鼠标落点附近，最后重置并随机生成独立的新 `id`。

## 三、后继规划：节点连线 (Connecting Mechanism)

随着当前快照与复制框架的落地，我们即将在此目录补充**连线绘制与规则验证**流水线（此部分尚未完成，这是占位说明）：

1. **拖拽连线生成**：通过 `isValidConnection` 阻断特定输入/输出数据类型不匹配的异常连线。
2. **连接捕捉 (Snap) 与动画边缘**：确保连线创建后调用 `onConnect`，此调用会立刻经过我们在 Zustand 中布置好的 `takeSnapshot()` 收集，提供立即可用的撤销能力。

> **说明**：所有画布级控制按钮（如悬浮撤销等）都应遵循毛玻璃融合设计（`backdrop-blur`），无边框干扰，强调操作而非占据工具栏空间。

## 四、右键菜单系统 (Context Menu System)

### 画布右键菜单 (Canvas Context Menu)
用户在画布空白区域右键时弹出，包含以下功能：
1. **粘贴节点** — 从系统剪贴板中粘贴之前复制的节点（与 `Ctrl+V` 共享逻辑），节点定位到右键点击位置。
2. **切换背景色** — 循环切换 4 种预设背景主题：网格 → 暖纸 → 石板 → 纯净。
3. **全屏模式切换** — 使用 `Fullscreen API` 进入/退出全屏模式；全屏中浏览器原生 `Esc` 键可退出，或再次右键选择"退出全屏"。

### 节点右键菜单 (Node Context Menu)
用户在节点上右键时弹出，以分割线分为四组：
1. **复制节点** — 将当前节点写入系统剪贴板（与 `Ctrl+C` 共享逻辑）。
2. **节点配置** / **重新执行** / **断开连接** — 预留功能（`disabled`），用于后续迭代。
3. **添加备注** / **导出数据** — 预留功能（`disabled`），用于后续迭代。
4. **删除节点** — 危险操作，红色高亮，删除前自动保存快照到历史栈。

### 实现文件
- `CanvasContextMenu.tsx` — 画布右键菜单组件 + `buildCanvasMenuItems()` 工厂函数
- `NodeContextMenu.tsx` — 节点右键菜单组件 + `buildNodeMenuGroups()` 工厂函数

## 五、全屏模式 (Fullscreen Mode)

基于浏览器 `Fullscreen API`（`element.requestFullscreen()`）实现：
- 进入方式：右键菜单 → "进入全屏"
- 退出方式：`Esc` 键（浏览器原生）或右键菜单 → "退出全屏"
- 状态同步：通过 `fullscreenchange` 事件监听器，确保 UI 状态与实际全屏状态双向同步。

## 六、背景主题 (Canvas Background Themes)

画布支持 4 种预设背景，通过右键菜单循环切换：

| Key    | 描述           | 暗色                | 亮色              |
|--------|---------------|--------------------|--------------------|
| grid   | 网格（默认）    | 深空 + 网格线       | 暖灰 + 网格线       |
| paper  | 暖纸质感        | `#1a1b1e`          | `#faf9f6`          |
| slate  | 石板灰          | `slate-900`        | `slate-100`        |
| clean  | 纯净无干扰      | 纯黑                | 纯白               |
