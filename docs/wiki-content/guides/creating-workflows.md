# 创建工作流

本文档详细介绍 StudySolo 工作流系统的核心概念与操作方法。

如果你的任务只需要一句答案，普通对话就够了。如果任务需要多轮处理、多个中间结果或可重复执行，就应该使用工作流。

| 任务类型 | 推荐方式 |
| --- | --- |
| “解释一下什么是梯度下降” | AI 对话 |
| “帮我制定 6 周机器学习入门计划” | 工作流 |
| “把这篇论文变成阅读笔记、术语表和复盘题” | 工作流 |
| “根据我的错题生成复习安排” | 工作流 |

## 什么是工作流

工作流是由多个 AI 节点组成的学习路径，每个节点负责一个具体的学习任务。节点之间通过连线形成完整的学习规划流程。

### 工作流生命周期

```
输入目标 → AI 分析 → 知识拆解 → 路径规划 → 资源推荐 → 输出计划
```

更完整的生命周期通常是：

```text
定义目标 → 拆解任务 → 配置节点 → 连接流程 → 运行检查 → 迭代优化
```

每个阶段都有不同关注点：

- 定义目标：明确学习主题、时间、基础和输出要求
- 拆解任务：把大目标拆成可执行步骤
- 配置节点：为每一步设置输入、提示词和输出格式
- 连接流程：让数据按正确顺序流动
- 运行检查：查看节点状态和中间结果
- 迭代优化：根据结果调整节点或连接

## 设计一个好目标

工作流质量首先取决于目标描述。一个好目标应该包含：

- 学习对象：要学什么
- 当前基础：已经会什么，不会什么
- 时间约束：多久完成，每天能投入多久
- 结果形式：要计划、笔记、题目、项目，还是复盘清单
- 评价标准：怎样算完成，怎样算学会

示例：

```text
我想在 6 周内学习机器学习基础。
我会 Python 和基础线性代数，但没有系统学过模型评估。
请生成一个每周学习计划，每周包含核心概念、代码练习、复盘问题和一个小项目。
最终输出用表格展示，并标注每周的验收标准。
```

## 节点类型详解

### 基础节点

| 类型 | 说明 | 用途 |
|------|------|------|
| **用户输入** | 接收用户的学习目标 | 工作流起点 |
| **AI 分析** | 分析知识点结构 | 理解学习需求 |
| **AI 规划** | 生成学习路径 | 制定学习计划 |
| **资源推荐** | 推荐学习资料 | 提供学习资源 |
| **总结归纳** | 输出学习摘要 | 整理学习要点 |

### 高级节点

- **条件分支**：根据用户水平调整路径
- **循环节点**：支持重复练习场景
- **评估节点**：检测学习效果

## 推荐工作流模板

### 学习计划模板

适合课程、技能、证书和长期目标：

```text
目标输入 → 背景分析 → 知识拆解 → 阶段规划 → 资源推荐 → 练习设计 → 输出计划
```

### 论文阅读模板

适合阅读学术论文、技术报告或长文档：

```text
材料输入 → 结构摘要 → 关键概念提取 → 方法分析 → 贡献与局限 → 复盘问题 → 阅读笔记
```

### 考试复习模板

适合期末、考研、证书或面试准备：

```text
考试目标 → 考点拆解 → 薄弱点评估 → 复习计划 → 题目生成 → 错题复盘 → 冲刺清单
```

## 创建方式

### 方式一：AI 自动生成（推荐）

1. 点击「新建工作流」
2. 输入学习目标描述
3. 点击「生成工作流」
4. AI 自动构建完整节点图

生成后不要急着运行，先检查这几项：

- 是否有明确的输入和输出
- 中间节点是否按合理顺序排列
- 是否有节点承担了过多任务
- 是否缺少评估、复盘或总结环节
- 资源推荐是否符合语言、难度和数量要求

如果结构大体合理，只需要局部修改。常见调整指令包括：

```text
把“生成学习计划”节点拆成“周计划”和“每日任务”两个节点。
```

```text
在最终输出前增加一个评估节点，检查计划是否过密、是否缺少复盘。
```

### 方式二：手动搭建

1. 从节点库拖拽所需节点
2. 配置每个节点的参数
3. 连接节点形成流程
4. 保存并运行测试

手动搭建时建议先放输入节点和输出节点，再补中间的分析、生成和评估节点。这样能确保流程始终围绕最终结果展开。

## 工作流编辑技巧

### 画布操作

- **拖拽**：移动节点位置
- **连线**：点击节点边缘，拖拽到目标节点
- **缩放**：鼠标滚轮或双指捏合
- **删除**：选中节点按 Delete 键

### 节点配置

每个节点支持以下配置：

- **输入参数**：定义节点接收的数据
- **输出格式**：设置节点返回的结构
- **AI 模型**：选择执行任务的模型
- **超时设置**：控制节点执行时间

## 节点设计原则

### 单一职责

一个节点只做一类任务。例如“分析基础”和“生成学习计划”最好分开，不要放在同一个节点里。

### 输入明确

节点需要知道自己处理什么材料。可以把前置节点输出、用户目标和格式要求写清楚。

### 输出可检查

输出最好有结构，例如表格、清单、分段标题或 JSON。结构化输出更容易被后续节点继续使用。

### 关键处加评估

在重要输出后增加评估节点，可以提前发现问题。例如学习计划生成后，检查是否过密、是否缺少实践、是否与时间约束冲突。

## 运行与调试

### 运行工作流

点击「运行」按钮，系统将：

1. 按拓扑顺序执行节点
2. 显示每个节点的执行状态
3. 输出最终结果

### 调试技巧

- 查看节点输出日志
- 使用「单步执行」模式
- 检查节点间数据传递

常见问题和处理方式：

| 现象 | 可能原因 | 处理方式 |
| --- | --- | --- |
| 最终结果很泛 | 前置目标不够具体 | 补充基础、时间和输出要求 |
| 计划不可执行 | 缺少评估节点 | 增加可行性检查 |
| 输出格式混乱 | 节点提示词太自由 | 明确表格、清单或模板 |
| 某个节点耗时过长 | 任务太大 | 拆成多个小节点 |
| 后续节点理解错 | 前置输出不稳定 | 限定前置节点输出结构 |

## 最佳实践

- **从简单开始**：先用 AI 生成基础框架，再手动优化
- **模块化设计**：将复杂流程拆分为子工作流
- **测试覆盖**：运行前检查所有节点配置
- **文档注释**：为复杂节点添加说明

一个成熟工作流不是一次生成出来的，而是在真实使用中逐步调出来的。先让它可运行，再让它更准确，最后再追求复用和自动化。
