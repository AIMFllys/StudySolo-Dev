# 使用节点

节点决定工作流每一步要做什么。合理使用节点，可以把复杂学习任务拆成清晰、可复用、可调试的步骤。

如果工作流是一条学习生产线，节点就是其中的工位。每个工位只处理一类任务，最后把结果交给下一个工位。

## 节点的基本组成

一个节点通常包含以下信息：

| 组成 | 说明 | 示例 |
| --- | --- | --- |
| 输入 | 节点要处理的内容 | 学习目标、论文、课程目录 |
| 指令 | 节点要完成的任务 | 拆解知识点、生成练习题 |
| 配置 | 模型、格式、限制条件 | 输出表格、使用中文、控制长度 |
| 输出 | 传递给后续节点的结果 | 计划、摘要、评估意见 |

节点配置越清楚，工作流越稳定。不要指望后续节点自动猜测前面节点的意图。

## 添加节点

在工作流画布中，你可以从节点库添加节点，也可以通过 AI 对话让系统自动生成节点。

添加节点时先确认三个问题：

- 这个节点要解决什么具体任务
- 它需要哪些输入
- 它应该输出什么结果

如果一个节点的职责说不清，通常说明它太大或不必要。

## 常见节点类型

### 输入节点

输入节点负责收集原始信息。它可以是一个学习目标、一段文本、一份课程目录，也可以是用户补充的背景。

好的输入示例：

```text
目标：4 周内复习概率论期末考试
基础：会基础公式，但不会做综合题
时间：每天 1 小时，周末 2 小时
输出：每周计划、每日任务、练习题和复盘问题
```

### 分析节点

分析节点负责理解输入，而不是直接给答案。它常用于拆解目标、识别难点、判断前置知识。

适合的指令：

```text
请分析这个学习目标需要哪些前置知识、核心模块和常见难点。
输出分为“已具备基础”“需要补齐”“高风险难点”三部分。
```

### 生成节点

生成节点负责产出内容，例如计划、笔记、题目、摘要或项目任务。生成节点要尽量有格式要求，例如用表格输出、每项包含验收标准、每个概念配一个例子。

### 评估节点

评估节点用于检查输出质量。它不一定改变内容，但能指出问题并给出修改建议。

适合检查：

- 学习计划是否过密
- 任务是否可执行
- 是否缺少复盘和练习
- 输出是否符合格式
- 内容难度是否匹配当前基础

### 输出节点

输出节点负责把前面的结果整理成最终版本。它应该追求清晰、完整、方便复制或阅读。

## 连接节点

节点之间通过连线传递数据。前一个节点的输出会成为后一个节点的上下文或输入材料。

连接节点时建议保持流程清晰：

- 从输入节点开始
- 把分析类节点放在生成类节点之前
- 把评估类节点放在生成结果之后
- 避免无意义的交叉连线

推荐连接方式：

```text
输入 → 分析 → 生成 → 评估 → 修订 → 输出
```

不推荐把所有节点都直接连到最终输出。这样会让上下文混乱，也不利于定位问题。

## 配置节点

不同节点会有不同配置项。常见配置包括输入内容、输出格式、模型偏好和执行说明。

```text
请按初学者视角解释，每个概念给出一个例子，并标注常见误区。
```

比起宽泛的“帮我总结”，明确的输入通常会得到更稳定的输出。

更完整的提示词可以这样写：

```text
请从初学者视角总结这段内容。
输出包括：核心概念、关键公式、常见误区和 3 道自测题。
每个核心概念都要给一个生活化例子。
```

## 调试节点

如果节点输出不符合预期，优先检查节点输入和提示词，而不是直接重建整个工作流。

常见调整方式：

- 补充背景信息
- 限定输出格式
- 拆分过大的任务
- 增加评估或总结节点
- 删除没有贡献的中间节点

| 问题 | 优先检查 |
| --- | --- |
| 输出跑题 | 输入节点和分析节点 |
| 内容太浅 | 分析节点是否缺少背景 |
| 计划不可执行 | 评估节点是否缺失 |
| 后续结果混乱 | 前置节点输出格式 |
| 运行很慢 | 是否有节点任务过大 |

## 节点拆分技巧

如果一个节点同时做了多件事，建议拆分。

例如，把这个节点：

```text
分析我的学习目标，生成 4 周计划，推荐资源，并设计练习题。
```

拆成四个节点：

1. 目标分析
2. 4 周计划生成
3. 资源推荐
4. 练习题设计

拆分后，每一步都能单独检查，后续也更容易复用。

## 节点命名建议

好的命名能让画布更容易阅读。

- 使用动词加对象：分析学习目标、生成周计划、评估可行性
- 避免泛化名称：处理、总结、AI 节点
- 对重复节点加阶段：第 1 周练习生成、第 2 周练习生成
- 对关键节点加目的：输出最终复习清单、检查计划过密风险

## 常用节点组合

### 计划生成组合

```text
目标输入 → 学情分析 → 知识拆解 → 周计划生成 → 可行性评估 → 最终计划
```

### 笔记整理组合

```text
材料输入 → 章节摘要 → 概念提取 → 例题生成 → 复盘问题 → 笔记输出
```

### 练习强化组合

```text
错题输入 → 错因分析 → 知识点定位 → 同类题生成 → 答案解析 → 复习建议
```

把节点当成可组合的积木，而不是一次性聊天内容。这样你会更容易搭出稳定、有复用价值的学习流程。
