# 核心概念

StudySolo 用工作流组织学习任务。你可以把一个学习目标拆成多个节点，让 AI 逐步完成分析、规划、生成、评估和总结。

它和普通聊天最大的区别是：普通聊天更像一次问答，工作流更像一套可保存、可运行、可复用的学习流程。你可以不断调试节点，让它变成自己的学习方法模板。

## 工作流

工作流是一组按顺序或条件连接的节点。每个工作流对应一个明确的学习任务，例如准备考试、拆解论文、规划课程或整理复习资料。

一个典型工作流包含：

- 输入：学习目标、材料、约束条件或已有基础
- 处理：分析、拆解、检索、生成、评估
- 输出：学习计划、笔记、题目、摘要或下一步行动

| 场景 | 工作流可以做什么 |
| --- | --- |
| 考试复习 | 拆解考点、安排复习周期、生成测验 |
| 论文阅读 | 提取摘要、概念、方法、结论和可追问问题 |
| 技能学习 | 规划阶段、推荐练习、设计项目任务 |
| 课程整理 | 把目录变成笔记结构、复盘清单和学习路径 |

## 节点

节点是工作流里的最小执行单元。每个节点负责一类具体任务，并把结果传递给后续节点。

常见节点包括：

- 输入节点：接收用户目标或学习材料
- AI 分析节点：拆解主题、识别难点、提取关键概念
- 内容生成节点：生成大纲、摘要、卡片或练习题
- 逻辑控制节点：根据条件选择不同路径
- 输出节点：保存、导出或展示最终结果

> 一个好节点应该像一张清楚的任务卡片：输入是什么、要做什么、输出什么，都能一眼看出来。

## 连接线

连接线表示数据从一个节点传到另一个节点。前一个节点的输出，会成为后一个节点的上下文。

例如：

```text
学习目标 → 目标分析 → 知识拆解 → 周计划 → 每日任务
```

这条连接关系意味着「周计划」不是凭空生成的，而是基于前面的目标分析和知识拆解。连接线越清晰，最终结果越稳定。

## 画布

画布用于编辑工作流结构。你可以在画布上添加节点、拖拽位置、连接节点，并查看运行时的状态变化。

画布关注的是流程结构：

- 哪个节点先执行
- 哪些节点需要依赖前置结果
- 哪些节点可以并行处理
- 最终结果从哪里输出

当工作流变复杂时，建议把同类节点放在相邻位置。例如把分析节点集中在前半段，把生成和评估节点放在后半段。

## AI 对话

AI 对话用于把自然语言意图转成工作流操作。你可以直接描述想做的事，例如“帮我把这篇论文拆成复习计划”，系统会尝试生成或调整对应的节点图。

AI 对话适合处理：

- 从零创建工作流
- 修改已有节点
- 解释某个节点的输出
- 根据运行结果继续优化学习计划

不适合只说一句非常模糊的请求，例如“帮我学习英语”。更好的写法是：

```text
我想在 8 周内提升英语阅读能力，目标是能读懂技术文档。
我每天可以学习 40 分钟，词汇量大约 5000。
请生成一个包含阅读材料、词汇复盘和每周测验的工作流。
```

## 运行记录

运行记录用于观察每个节点实际输出了什么。它是调试工作流最重要的信息来源。

如果最终结果不好，优先检查：

1. 输入节点是否描述清楚
2. 分析节点是否抓住关键约束
3. 中间节点是否过大或过泛
4. 输出节点是否有明确格式要求

## 推荐心智模型

你可以用下面这句话理解 StudySolo：

> 聊天负责表达意图，画布负责组织流程，节点负责执行任务，运行记录负责帮助你调试。

当你不知道该改哪里时，先找到输出不理想的最早节点，再从那里向前检查输入和连接关系。
