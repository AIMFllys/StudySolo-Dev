# Issue 管理规范

> 最后更新：2026-04-09

## 1. 全员 Issue 提交义务

每位团队成员在开发过程中发现的任何问题，**尤其是安全漏洞**，必须立即在 GitHub Issues 中提交。

## 2. Issue 标签体系

| 标签 | 优先级 | 说明 |
|------|--------|------|
| `🔴 security` | **P0** | 安全漏洞（RLS 缺失、权限泄露、注入风险等） |
| `🟠 bug` | P1 | 功能性 Bug |
| `🟡 enhancement` | P2 | 功能改进建议 |
| `🔵 docs` | P3 | 文档问题 |
| `⚪ question` | — | 讨论与提问 |

## 3. 安全 Issue 模板

```markdown
## 🔴 安全问题报告

**严重程度：** Critical / High / Medium / Low

**问题描述：**
<!-- 简述安全漏洞 -->

**影响范围：**
- 受影响文件：
- 受影响接口：
- 受影响用户范围：

**复现步骤：**
1.
2.

**修复建议：**
<!-- 如有思路请提供 -->
```
