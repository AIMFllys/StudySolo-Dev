## 1. S4.1 Wiki 初始化骨架

**不做**：

### 1.2 目标结构

```
frontend/src/app/(wiki)/
├── wiki/
│   ├── layout.tsx              # Wiki 专属布局（不继承主应用 sidebar）
│   ├── page.tsx                # Wiki 首页（文档列表）
│   └── [...slug]/
│       └── page.tsx            # 动态文档页（Markdown 渲染）

docs/wiki-content/
├── getting-started/
│   └── quick-start.md          # 占位：快速开始
├── guides/
│   └── creating-workflows.md   # 占位：创建工作流
└── README.md                   # Wiki 内容目录说明
```

### 1.3 Wiki Layout 要点

- 独立于主应用的 sidebar 布局
- 左侧：文档导航（从 `docs/wiki-content/` 目录结构生成）
- 右侧：文档内容区
- 不继承 `DashboardShell`

### 1.4 Markdown 渲染

- 使用 `react-markdown` 或 `next-mdx-remote`（项目已有 `react-markdown` 依赖）
- 代码高亮使用项目已有的 Shiki
- 不引入额外的 MDX 依赖

## 2. S4.2 CI 增强

### 2.1 依赖方向检查

在 CI 中添加后端依赖方向检查：

```yaml
# 添加到 ci-backend.yml
- name: Check dependency direction
  run: |
    # nodes/ 不应该 import services/
    if grep -r "from app.services" backend/app/nodes/ --include="*.py" | grep -v __pycache__; then
      echo "VIOLATION: nodes/ imports services/"
      exit 1
    fi
    # api/ 不应该 import engine/
    if grep -r "from app.engine" backend/app/api/ --include="*.py" | grep -v __pycache__; then
      echo "VIOLATION: api/ imports engine/"
      exit 1
    fi
```

**注意**：当前已知 `nodes/` 中存在 4 处对 `services/` 的反向依赖（问题 13.2.3）。这些需要先标记为已知例外，或在 CI 中排除：

# - backend/app/nodes/input/knowledge_base/node.py

### 2.2 前端裸 fetch 回归检查

```yaml
# 添加到 ci-frontend.yml
- name: Check no raw fetch in features
  run: |
    if grep -rn "await fetch(" frontend/src/features/ --include="*.ts" --include="*.tsx" | grep -v node_modules; then
      echo "VIOLATION: Raw fetch() found in features/"
      echo "Use authedFetch or authedStreamFetch instead"
      exit 1
    fi
```

### 2.3 Agent 测试基线保护

```yaml
# 更新 ci-agents.yml
- name: Test
  run: |
    result=$(pytest agents/${{ matrix.agent }}/tests/ -q 2>&1)
    echo "$result"
    # 确保 code-review-agent 至少 177 passed
    if [[ "${{ matrix.agent }}" == "code-review-agent" ]]; then
      passed=$(echo "$result" | grep -oP '\d+ passed' | grep -oP '\d+')
      if [ "$passed" -lt 177 ]; then
        echo "REGRESSION: Expected at least 177 passed, got $passed"
        exit 1
      fi
    fi
```

### 2.4 验收标准
