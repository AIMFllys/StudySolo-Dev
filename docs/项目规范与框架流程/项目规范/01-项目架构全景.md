## 一、项目定位

StudySolo 是 1037Solo 生态中的 AI 学习工作流平台。用户在前端画布中编排学习节点，后端以 DAG 执行引擎驱动节点运行，通过 SSE 把执行状态、增量输出与完成结果实时回推给前端，同时将知识库、用量账本、工作流运行记录等数据写入 Supabase。

当前仓库是一个多语言单仓工程，包含：

- `frontend/`：主前端，Next.js App Router
- `backend/`：主后端，FastAPI
- `agents/`：子后端 Agent 服务目录
- `introduce/`：独立产品介绍页，Vite 静态 SPA
- `shared/`：跨项目共享模块，当前以 Git submodule 接入
- `supabase/`：数据库迁移与基线

### 2.1 子项目角色

| 目录 | 当前角色 | 备注 |
|------|----------|------|
| `frontend/` | 主用户端与后台管理端 | Next.js 16.1.6，App Router |
| `backend/` | 统一 API、执行引擎、AI 路由、账本与知识库 | FastAPI 0.115.12 |

### 2.2 `shared/` 接入说明

- 本仓库中的 `shared/` 通过 `.gitmodules` 以 **Git submodule** 接入。
- Platform Monorepo 中的 StudySolo 同步是 subtree 语义，这是另一件事，不能混写。

## 七、节点系统与执行链

### 7.1 节点类型

权威来源：`frontend/src/types/workflow.ts` + `backend/app/nodes/**`

当前共有 25 种节点类型：

- 输入层：`trigger_input`、`knowledge_base`、`web_search`
- 分析/控制层：`ai_analyzer`、`ai_planner`、`logic_switch`、`loop_map`
- 生成层：`outline_gen`、`content_extract`、`summary`、`flashcard`、`quiz_gen`、`mind_map`、`compare`、`merge_polish`
- 交互层：`chat_response`
- 输出层：`export_file`、`write_db`
- 结构层：`loop_group`
- 社区层：`community_node`
- Agent 层：`agent_code_review`、`agent_deep_research`、`agent_news`、`agent_study_tutor`、`agent_visual_site`

其中：

- 内置节点 24 种
- 社区扩展节点 1 种（`community_node`）

### 7.2 节点注册机制

- 后端通过 `backend/app/nodes/__init__.py` 自动发现并导入所有 `*.node` 模块。
- `BaseNode.__init_subclass__` 自动注册节点类型。
- `GET /api/nodes/manifest` 直接返回后端节点 manifest。
- 前端通过 `frontend/src/services/node-manifest.service.ts` 获取 manifest 并缓存。

### 7.3 执行与 SSE

当前前后端共同使用的工作流 SSE 事件类型包括：

- `node_status`
- `node_input`
- `node_token`
- `node_done`
- `node_progress`
- `loop_iteration`
- `workflow_status`
- `heartbeat`
- `workflow_done`
- `save_error`

### 10.2 Gateway 接入状态

当前 `backend/config/agents.yaml` 已注册：

```yaml
agents:
  code-review:
    ...
  deep-research:
    ...
  news:
    ...
  study-tutor:
    ...
  visual-site:
    ...
```

因此当前事实是：

- 5 个子后端 Agent 已进入 Gateway 注册表
- 前端节点商店已新增 `Agent` 分区
