# 2026-04-12 更新日志

## 1. Phase 4B：repo-aware 同序风格等价 identifier canonicalization 落地

今天继续沿着 Phase 4B 做一个 repo-aware 利用治理闭环：只收口 `code-review-agent` 的 shared identifier canonicalization 与连续窗口命中逻辑，不触碰 Gateway、不改公共 API、不引入本地仓库读取。

### 完成内容
1. **新增 repo-aware canonical identifier helper**
   - `agents/code-review-agent/src/core/upstream_review.py`
   - 新增：
     - `canonical_shared_identifier(...)`
     - `repo_aware_identifiers(...)`
   - 统一把 camelCase / PascalCase / snake_case / acronym 风格收口为 `lowercase_with_underscores`
   - 命中示例：
     - `requestBody -> request_body`
     - `HTTPClient -> http_client`
     - `apiURL -> api_url`
     - `userID -> user_id`
     - `errorCount -> error_count`
2. **shared identifier overlap 统一改走 canonical form**
   - `_shared_identifier_candidates(...)` 改为基于 canonical overlap 求交
   - generic token 过滤继续保留，但只过滤单词级 token
   - `request_body`、`status_code`、`error_count` 这类 compound token 保留
3. **连续窗口评分改为复用同一套 repo-aware identifier 真相**
   - `agents/code-review-agent/src/core/agent.py`
   - `_preview_forwarded_context(...)` 按行命中统计改为复用 `repo_aware_identifiers(...)`
   - 连续窗口评分公式不变，仍按：
     - 唯一 shared identifier 数
     - shared identifier 命中总次数
     - 命中的行数
     - 更早的窗口起点
4. **锁住顺序边界，不放宽到子词重排**
   - `cacheAdapter` 与 `adapter_cache`
   - `userID` 与 `id_user`
   - 以上都继续视为不重叠
5. **repo-aware prompt hints 值语义已统一**
   - 例如：
     - `renderbadge -> render_badge`
     - `totalcount -> total_count`
     - `formatcount -> format_count`
     - `handleerror -> handle_error`
     - `debuglog -> debug_log`
     - `statuscode -> status_code`
