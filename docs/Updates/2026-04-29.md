## 2. 测试失败修复

### 2.1 extract_json 正则贪婪匹配

- **文件**: `backend/app/services/workflow_generator.py`
- **问题**: `extract_json()` 的正则 `r"```(?:json)?\s*([\s\S]+?)```"` 在 JSON 值包含反引号时会提前截断
- **修复**: 改为 `r"```(?:json)?\s*\n([\s\S]+?)\n\s*```"`，要求 fence 前后有换行符

### 2.2 测试隔离 — 假节点污染 BaseNode._registry

- **文件**: `backend/tests/test_workflow_control_flow_property.py`
- **问题**: `EchoNode`、`BranchingNode`、`StreamingNode`、`FailingNode` 继承 `BaseNode` 时，`__init_subclass__` 自动将它们注册到全局 `_registry`，覆盖了真实的 `SummaryNode` 和 `LogicSwitchNode`（display_name 变为空字符串）
- **修复**: 给所有假节点添加 `_abstract = True`，跳过自动注册

### 2.3 测试隔离 — JWT_SECRET LRU 缓存泄漏

- **文件**: `backend/tests/test_auth_security_hardening_property.py`
- **问题**: `test_cors_property.py` 修改 `os.environ["JWT_SECRET"]` 后，`get_settings()` 的 `@lru_cache` 缓存了错误的值，导致后续 auth 测试 JWT 验证失败
- **修复**: 在测试中使用 `monkeypatch.setenv` + `get_settings.cache_clear()`

## 3. BUG-001 修复：实现 try_parse_json

- **文件**: `backend/app/nodes/_mixins.py`
- **方法**: `JsonOutputMixin.try_parse_json(raw_output) -> dict | list | None`
- **策略**: 3 级 fallback（直接解析 → 去除 markdown fence → 提取首个 JSON 对象/数组）
- **特性**: 同步方法，解析失败返回 `None`（不抛异常），与 `validate_json`（async，抛异常）互补
- **影响**: `LogicSwitchNode` 和 `LoopMapNode` 的 `post_process` 不再崩溃
- **测试**: 移除 10 个 BUG-001 skip 标记，全部恢复为 PASSED
- **关联**: 关闭 GitHub Issue #26
