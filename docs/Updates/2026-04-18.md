# 2026-04-18 更新日志（AI 智能体重命名修复 · 工具调用卡片布局优化）

## 1. 修复：智能体重命名后侧边栏不刷新

**问题**：通过 AI 智能体对话重命名工作流时，后端数据库更新成功，前端显示"完成"，但侧边栏工作流列表名称不变。

**根因**：`rename_workflow_tool` 返回的 `ToolResult` 没有携带 `ui_effect`，而前端 `handleAgentEvent` 只处理 `canvas_mutation` 和 `ui_effect` 两种事件。侧边栏的 workflows 列表来自 Server Component props，只有 `router.refresh()` 才能触发 Next.js 重新获取服务端数据。

**修复**：
- 后端 `rename_workflow_tool` 成功时返回 `ui_effect=UIEffect(type="router_refresh")`
- 前端 `handleAgentEvent` 新增对 `router_refresh` 类型的处理，收到后调用 `router.refresh()`

涉及文件：
- `backend/app/services/ai_chat/tools/workflow_tools.py`
- `frontend/src/features/workflow/hooks/use-stream-chat.ts`

## 2. 优化：工具调用卡片布局

**问题**：工具调用卡片中工具名被 `truncate` 截断为 "add..."，参数摘要反而占据更多空间；状态标签 "完成" 被挤成竖排；整体卡片偏高。

**修复**：
- 工具名使用 `shrink-0 whitespace-nowrap` 确保完整显示
- 参数摘要使用 `truncate min-w-0` 在空间不足时优先被截断
- 状态标签使用 `shrink-0 whitespace-nowrap` 防止换行
- 减小垂直间距：外层 `my-1.5` → `my-0.5`，按钮 `py-1.5` → `py-1`，`gap-2` → `gap-1.5`
- 圆角从 `rounded-lg` 改为 `rounded-md`，边框从 `border-[1.5px]` 改为 `border`

涉及文件：
- `frontend/src/components/layout/sidebar/agent-segments/ToolCallSegment.tsx`
