# 2026-03-27 更新日志

## 【面向用户】

### v0.8.0 品牌色回归、执行面板重构与知识库内化
1. **AI 模型选择器体验修补**：
   - **品牌色全量回归**：修复了所有模型均显示为相同颜色的问题。现在 DeepSeek、通义千问、智谱清言、豆包、Kimi 等模型均拥有各自独特的品牌视觉色，大幅提升辨识度。
   - **新增 7 款旗舰模型**：目录中正式补全了 GLM-4.5/5、豆包 Seed 2.0、Kimi K2.5 等最新模型，支持更多维度的推理与创作。
   - **加载稳定性提升**：新增了加载骨架屏与重试机制，彻底消除了由于网络波动导致的模型列表“空白”现象。
2. **AI 生成与画布布局进化**：
   - **智能自动排版**：修复了 AI 生成工作流时节点重叠的问题。现在生成的节点会按 340px*220px 的标准跨度整齐排列。
   - **强制入口保障**：所有生成的工作流现在都会自动补齐或强制设为以“用户输入”为起点，确保流程逻辑闭环。
3. **工作流执行面板（Redesign）**：
   - **多线路 Tab 筛选**：针对复杂分支流程，执行面板现已支持按“线路 1”、“线路 2”等独立 Tab 进行过滤，您可以精准追踪特定分支的运行结果。
   - **执行一致性保障**：现在执行工作流时会直接采用您在画布上最新的修改，无需等待云端同步。
   - **线段化时间轴视觉**：优化了运行步骤的连接线视觉，实时状态（Running/Skipped/Error）更易读。
4. **即时配置 Popover**：
   - 节点配置不再占用繁重的右侧抽屉，改为就近锚点的弹窗（Popover）模式。修改即刻生效，执行开始时自动关闭，操作链路显著缩短。
5. **知识库节点“全功能”内化**：
   - **节点内操作优先**：原“知识库管理”独立页面正式退役。现在，您可以在“知识库”节点内直接完成文件上传、处理状态监控、文档摘要阅读及分块预览，真正实现“在节点内完成所有工作”。

## 【面向开发者】

### v0.8.0 链路重构、Manifest 驱动与 SOP 规格基线
1. **执行链路重构（POST 改版）**：
   - **内存执行态**：执行接口从 `GET` 切换为 `POST`，由前端直接提交内存中的 `nodes/edges`。彻底解决了“执行读取数据库旧图”产生的竞态问题。
   - **SSE 数据解析优化**：引入了基于 `ReadableStream` 的流式解析 helper，不再依赖不稳定的 `EventSource`。
2. **节点能力 Manifest 体系落地**：
   - **数据驱动配置**：后端 `/api/nodes/manifest` 现在提供完整的 `config_schema`。前端配置弹窗改为由该 Schema 动态生成，彻底解耦了硬编码表单。
   - **动态选项支持**：新增 `dynamic_options` 机制，支持在配置节点时异步拉取用户侧上下文（如知识库文档列表）。
3. **架构基线与 SOP 纠偏**：
   - **工程事实源统一**：重写了 `workflow_node_system_analysis.md`，明确以真实代码基线为准。
   - **能力规格表**：建立了 `workflow_node_capability_spec.md` 审计框架，为后续 A/B 型节点的自动化测试与补全提供了唯一标准。
4. **后端健壮性巡检**：
   - **Lifespan SKU 校验**：实装了 FastAPI 启动时的配置巡检机制，自动发现并告警 `config.yaml` 与数据库 SKU 记录的偏移，杜绝静默失败。
   - **知识库 Headless 化**：`features/knowledge` 已重构为无界面的工具箱模式，专门服务于节点内交互组件。
