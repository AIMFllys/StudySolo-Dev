# 2026-03-17 更新日志

## 【面向用户】

### v0.3.5 工作流同步机制全面升级
1. **可靠的云端同步**：修复了在编辑工作流时，本地改动偶尔无法同步到云端的严重问题。现在您的每一步编辑都将被精准记录，重新进入页面可百分之百恢复。
2. **手绘感画布加载动画**：在工作流初始化期间，新增了精美的“电路板”手绘风格线条动画，取代了之前的空白等待。
3. **节点加载骨架屏**：为所有节点引入了“纸质感”骨架屏（Shimmer）效果，加载过程更丝滑，不再有生硬的闪烁。
4. **AI 生成状态占位**：当 AI 正在思考并生成工作流时，画布中央会显示一个手绘风格的加载节点，让您实时感知生成进度。

## 【面向开发者】

### v0.3.5 工作流同步架构重构 (Periodic Snapshot)
1. **从 Debounce 转向 Interval Snapshot**：废弃了依赖 `isDirty` 副作用的传统限流同步，改用每 2s 轮询快照 hash 差异的机制。通过 `setInterval` 脱离 React Lifecycle 限制，彻底解决了连续修改导致的遗漏更新问题。
2. **前后端执行锁与竞态修复**：引入了执行锁机制。在后端通过 SSE 推送执行结果期间，前端会自动暂停定时快照上传，避免后端写入的执行状态被前端旧快照覆盖。
3. **IndexedDB 本地先行策略**：同步链路改为“实时写本地数据库 (localforage) -> 后台限流 (8s) 上传云端”，保证了即便浏览器崩溃，本地缓存也能在下次进入时作为“脏数据”优先恢复并补传云端。
4. **GeneratingNode 状态机**：新增独立节点类型用于承载 AI 生成中的中间态，优化了大量节点注入 React Flow 时的首帧渲染压力。
