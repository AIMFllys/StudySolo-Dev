# 2026-03-26 更新日志

## 【面向用户】

### v0.7.0 计费分析仪表盘、循环容器与协作体系完备
1. **AI 账单与仪表盘系统**：
   - **个人账单中心**：新增个人仪表盘，实时展示您的 AI 使用量与累计成本（以 CNY 计）。
   - **智能模型目录**：模型选择器升级为从实时目录拉取。现在，您可以清楚地看到每个模型的实时价格、性能级别以及解锁所需会员等级。
2. **工作流画布交互大升级**：
   - **连线语义强化**：连线上现在可以直接显示“分支标签”和“等待时长”（⏱）。点击等待图标即可快速修改延迟执行间距。
   - **真“容器”级循环块**：节点现在可以直接拖入循环组（Loop Group）内进行嵌套。拖入即自动绑定，拖出即释放。子节点运行进度（如“第 2/5 轮”）实时可见。
   - **逻辑分支视觉强化**：分支节点改为醒目的琥珀色视觉，并增加了从侧边拖出新分支的交互引导。
3. **协作与分享中心**：
   - **读写分离路由**：预览分享页（`/s/[id]`）与编辑页（`/c/[id]`）正式分离，防止误触导致数据污染。
   - **多人协作控制**：支持通过邮箱邀请协作者，并可分配查看者（Viewer）或编辑者（Editor）权限。
   - **实时协作感知**：在画布顶部新增了协作者头像序列，利用不同颜色标识在线协作人员，工作进度一目了然。
4. **节点 UI 精细化排布**：
   - **闲置态展示**：未运行节点新增“闲置中”（🕒）状态占位，节点布局更加工整稳定。
   - **按需显示模型**：仅 AI 处理类节点会显示模型选择器，工具类节点（如写库、搜索）界面更清爽。

## 【面向开发者】

### v0.7.0 计费账本、控制流收口与 RLS 协作架构
1. **CNY 计费基座与双层目录**：
   - **Catalog 体系升级**：引入 `ai_model_families`（逻辑族）与 `ai_model_skus`（物理计费点）表结构。
   - **Usage Ledger**：实现了分钟级聚合的 `ss_ai_usage_minute` 账本，支持多厂商差异化扣费逻辑。
   - **Backend Service**：新增 `ai_catalog_service.py` 进程内缓存目录，提升了 90% 的模型元数据读取性能。
2. **Workflow Canvas 控制流收口**：
   - **连线类型唯一化**：废弃了所有特殊 Edge 类型，全量收口为 `sequential`。
   - **父子坐标映射**：重构了 `loop-group-drop.ts` 纯算法层，利用 React Flow 的 `extent: 'parent'` 实现了稳定的嵌套节点坐标变换与解绑逻辑。
   - **分支管理组件**：实装 `BranchManagerPanel.tsx`，通过 Zustand 拦截器保障了分支命名的唯一性与自增逻辑。
3. **协作鉴权与 RLS 策略**：
   - **协作关联表**：新建 `ss_workflow_collaborators` 表，配合 Supabase RLS 策略实现了行级数据隔离。
   - **核心鉴权依赖**：在 FastAPI 后端提供 `check_workflow_access()` 依赖项，三级权限（Owner/Editor/Viewer）在逻辑层强制校验。
4. **UI 健壮性修复**：
   - **requiresModel 元数据**：在 `workflow-meta.ts` 引入静态 flag，替代了脆弱的字符串过滤。
   - **缓存隔离**：修复了 `useWorkflowCatalog` 的模块级缓存污染问题，现在缓存按 `userId + tier` 进行键控隔离，保障了多租户场景的安全。
